Installing Vue

Context: The aim of Monday Club is to develop some apps, learning some software/code in the process and spending some time with my friend on a productive project. The initial testing app is called 'app-1'. Today I installed the software I need. The installation on Windows 10 ran into issues (I think they are related to user permissions/path, I will revisit them when I have time). I switched to Linux Ubuntu 20.04 and the install took about 10 minutes. 

# Install nodejs
$  curl -sL https://deb.nodesource.com/setup_14.x | sudo -E bash -
$  sudo apt-get install -y nodejs
					
# Install vue using npm (node package manager)
$  sudo npm install -g @vue/cli
					
# navigate to project folder and clone project
$  cd ~/projects/web/
$  git clone <url/app_name>
$  cd <app_name>
					
# download and install dependencies
$  npm install

# start the server
$  npm run serve
Once the server is running, it will display the IP address and port required for access.
In my particular setup I'm going to run a development server that holds all my the repositories. I'll access it over SSH.
# install openssh if required
$  sudo apt-get install openssh-server

# check the process is running after install (look for sshd process)
$  ps aux | grep sshd

# any problems, then restart it
$  /etc/init.d/sshd restart

Vue Plugins

The Vue system uses various plugins. My understanding of these is very limited at the moment - a sentence on each essentially.

  • Babel - JavaScript compiler for converting ECMAScript 2015+ code into backwards compatible js for older environments.
  • Prettier - opinionated code formatter. It enforces a consistent style by parsing your code and re-printing it.
  • ESLint - Find and fix problems in your JavaScript code.
  • Router - official router for Vue.js, used for making Single Page Applications.
When creating a new Vue project there are more options of things to include, but for our basic 'hello, world' we are just looking at these options.
The repo includes a 'package.json' file which contains the names of the packages used in the project. Running npm install in the app project folder will parse this file and install the required packages used by the current project.

File Structure of Vue

Notes on the common files in a Vue project.

  • folder: node_modules [holds the packages that npm installs]
  • folder: public [the client side folder]
  • folder: src [folders for the vue files]
    • assets
    • components [components of the projects that are not main views, normally reusable]
    • router [routes for components]
    • store [part of vuex - not totally sure of use at this point]
    • views [components that are used in routes]
  • .broswerslistrc [list of browsers versions that project will allow]
  • .eslint.js [eslint file]
  • .gitignore [list of files that git will ignore]
  • babel.config.js [babel config]
  • package-lock.json
  • package.json [list of package dependencies for the project]
  • README.md [markdown readme for project]

These folders/files are not set in stone and can be altered to suit the project, etc.

DOM

The Document Object Model (DOM) is a cross-platform and language-independent interface that treats an XML or HTML document as a tree structure wherein each node is an object representing a part of the document. Vue allows declarative rendering of data to the DOM using a template syntax.

Interpolation

Interpolation is used to mean 'to alter or enlarge (a writing) by inserting new material'. In this case the HTML is being interpolated by outputted data from the JS. In the following example HTML the 'message' in the curly brackets is a placeholder for data from the JS. 

<div id="app">
  {{ message }}
<div>

The JS below targets the DOM element '#app' in the HTML above. In the JS console app.message can be changed and it will be updated immediately without refreshing the whole page, just that element of the DOM.

var app = new Vue({
  el: '#app',
  data: {
    message: 'Hello Vue!'
  }
})

Directives

Attributes prefixed with 'v-' e.g v-bind are directives. They apply special reactive beahaviour to the rendered DOM.
